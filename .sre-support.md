# tg-gateway — Support Runbook

Telegram gateway service used by all NexteraLabs apps to send messages, Supported and monitored by the SRE_Agent. Runs on port 8100.

- API: `http://<host>:8100`
- Health: `GET /channels`
- Container: `tg-gateway`
- Depends on: host PostgreSQL

## Known Issues

### Container starts but messages not delivered

- **Symptom**: `/channels` returns 200 but messages sent via `/send` silently fail; no Telegram delivery
- **Resolution**: The channel is registered but the bot token may be invalid or the bot was removed from the group. Check `docker logs tg-gateway` for `getUpdates` errors. Re-register the channel with a valid token: `POST /register-channel`.
- **Confidence**: ██████░░░░ 3/5 (asks briefly)

### API returns 500 on startup — database connection failed

- **Symptom**: `GET /channels` returns 500 or connection refused; logs show `asyncpg` connection error
- **Resolution**: Host PostgreSQL is not reachable from inside the container. Verify `DATABASE_URL_FROM_CONTAINER` in `.env` uses `host.docker.internal` (not `localhost`). Restart: `docker compose restart api`.
- **Confidence**: ████████░░ 4/5 (act, notify after)

### Duplicate polling conflict — bot already has active polling

- **Symptom**: Logs show `Conflict: terminated by other getUpdates request`
- **Resolution**: Another instance is polling the same bot token. Stop the duplicate: check for other containers or local processes running with the same token. Only one poller per bot token is allowed.
- **Confidence**: ██████████ 5/5 (autonomous)

### Callback not delivered to caller after button click

- **Symptom**: User clicks button in Telegram, prompt stays PENDING, caller never receives response
- **Resolution**: The `callback_url` is unreachable from inside the container. Verify the URL is accessible from the Docker network. For local dev use `host.docker.internal` not `localhost`.
- **Confidence**: ██████░░░░ 3/5 (asks briefly)

## Runbook

### Check registered channels

```bash
curl http://localhost:8100/channels
```

### Send a test message to a channel

```bash
curl -X POST http://localhost:8100/send \
  -H "Content-Type: application/json" \
  -d '{"channel_id": "mysecbrainchat", "text": "SRE test"}'
```

### Restart the container

```bash
docker compose restart api
```

### Environment variables required

```
DATABASE_URL_FROM_CONTAINER=postgresql+asyncpg://user:pass@host.docker.internal:5432/db
TELEGRAM_BOT_TOKEN=<default bot token for prompts>
TELEGRAM_TARGET_CHAT_ID=<default chat id>
```
