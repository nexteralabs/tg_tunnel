# Telegram MCP Gateway – Implementation Guide

## 1. Overview

The **Telegram MCP Gateway** is a local service that allows multiple AI assistants to communicate with Telegram channels via **assigned bots**.
It provides:

* Idempotent registration of channels (with associated bot and callback URL)
* Polling of Telegram channels for new messages
* Forwarding messages to the assistant’s callback with **retry logic**
* Notification in the channel if the assistant is offline
* Simple send API for assistants to respond

---

## 2. Concepts

| Term     | Description                                                    |
| -------- | -------------------------------------------------------------- |
| Channel  | Telegram chat/channel (chat_id) monitored by MCP gateway       |
| Bot      | Telegram bot token assigned to a channel for sending/receiving |
| Callback | Assistant endpoint that receives messages from the channel     |
| Endpoint | Channel + Bot + Callback registration                          |

**Key rule:** Each channel has **one bot assigned**. Registration is idempotent.

## 3. API Endpoints

### 3.1 Register Channel

```
POST /register-channel
{
  "channel_id": "finance_ai",
  "telegram_chat_id": "-1001234567890",
  "bot_token": "123456:ABCDEF...",
  "callback_url": "http://localhost:5001/on_telegram"

```

**Behavior:**

* Starts polling the channel
* Overwrites previous registration (idempotent)
* Returns: `{"status":"Channel finance_ai registered."}`

```

{
  "channel_id": "finance_ai",

**Behavior:** Sends a message via the bot assigned to the channel.

```

### 3.4 Unregister Channel

```
DELETE /channels/{channel_id}
```

Stops polling and removes registration.

## 4. Polling + Callback Logic

* Polling is **per channel**, async, with `getUpdates`.
* If the callback fails, retry **N times** with a delay.
* After all retries fail, send **notification in the channel** and stop retrying.

**Configurable constants:**

```python
MAX_RETRIES = 3
RETRY_DELAY = 5  # seconds
```

## 5. Python Implementation (FastAPI + Async)

```python
import asyncio
import httpx
from fastapi import FastAPI
from typing import Dict
import time

app = FastAPI()

# In-memory registry: channel_id -> {telegram_chat_id, bot_token, callback_url, last_update}
channels: Dict[str, dict] = {}

MAX_RETRIES = 3
RETRY_DELAY = 5  # seconds

# -------------------------
# Helper functions
# -------------------------
async def send_to_telegram(chat_id, bot_token, text):
    async with httpx.AsyncClient() as client:
        await client.post(
            f"https://api.telegram.org/bot{bot_token}/sendMessage",
            json={"chat_id": chat_id, "text": text},
        )

async def forward_to_callback(chat_id, bot_token, callback_url, message):
    for attempt in range(1, MAX_RETRIES + 1):
        try:
            async with httpx.AsyncClient(timeout=5) as client:
                res = await client.post(callback_url, json=message)
                res.raise_for_status()
            return True
        except Exception as e:
            print(f"[Attempt {attempt}] Callback failed for {chat_id}: {e}")
            if attempt < MAX_RETRIES:
                await asyncio.sleep(RETRY_DELAY)

    # All retries failed → notify channel
    notice = "⚠️ Assistant offline, could not deliver message."
    await send_to_telegram(chat_id, bot_token, notice)
    return False

async def poll_channel(channel_id):
    ep = channels[channel_id]
    chat_id = ep["telegram_chat_id"]
    bot_token = ep["bot_token"]
    callback_url = ep["callback_url"]
    last_update = ep.get("last_update", 0)

    print(f"[MCP] Polling started for channel {channel_id}")
    async with httpx.AsyncClient() as client:
        while channel_id in channels:
            try:
                resp = await client.get(
                    f"https://api.telegram.org/bot{bot_token}/getUpdates",
                    params={"offset": last_update + 1, "timeout": 30},
                )
                data = resp.json()
                for update in data.get("result", []):
                    last_update = update["update_id"]
                    ep["last_update"] = last_update

                    msg = update.get("message")
                    if not msg:
                        continue

                    message_event = {
                        "type": "telegram.message.created",
                        "channel_id": channel_id,
                        "telegram_chat_id": chat_id,
                        "from": msg["from"].get("username", "unknown"),
                        "text": msg.get("text", ""),
                    }

                    await forward_to_callback(chat_id, bot_token, callback_url, message_event)

            except Exception as e:
                print(f"[MCP] Polling error for {channel_id}: {e}")
                await asyncio.sleep(5)

# -------------------------
# API Endpoints
# -------------------------
@app.post("/register-channel")
async def register_channel(data: dict):
    channel_id = data["channel_id"]
    channels[channel_id] = {
        "telegram_chat_id": str(data["telegram_chat_id"]),
        "bot_token": data["bot_token"],
        "callback_url": data["callback_url"],
        "last_update": 0,
    }
    asyncio.create_task(poll_channel(channel_id))
    return {"status": f"Channel {channel_id} registered."}

@app.post("/send")
async def send_message(data: dict):
    channel_id = data["channel_id"]
    text = data["text"]
    ep = channels.get(channel_id)
    if not ep:
        return {"error": f"Channel {channel_id} not registered."}
    await send_to_telegram(ep["telegram_chat_id"], ep["bot_token"], text)
    return {"status": "sent"}

@app.get("/channels")
async def list_channels():
    return [{"channel_id": k, "telegram_chat_id": v["telegram_chat_id"]} for k,v in channels.items()]

@app.delete("/channels/{channel_id}")
async def unregister_channel(channel_id: str):
    if channel_id in channels:
        del channels[channel_id]
        return {"status": f"Channel {channel_id} unregistered."}
    return {"error": f"Channel {channel_id} not found."}
```

## 6. Features & Guarantees

* **Idempotent registration** → re-register safely after disconnect
* **Retry + notification** → avoids infinite retries if client offline
* **Per-channel polling** → multiple channels/bots supported
* **Simple API** → easy for assistants to integrate
* **Lightweight** → no DB or message queue required
